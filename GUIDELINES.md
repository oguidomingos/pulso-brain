# PULSO — Guia de Qualidade e Padrões

Documento de referência para garantir qualidade e consistência em todas as entregas do projeto Pulso.

---

## 1. ESTRUTURA DE PROCESSOS

### Fases de um Projeto Pulso

```
1. DISCOVERY (1-2 dias)
   ↓
2. DIAGNÓSTICO (2-3 dias)
   ↓
3. DESIGN (5-7 dias)
   ↓
4. BUILD (5-10 dias)
   ↓
5. CONTEÚDO (5-7 dias)
   ↓
6. REVIEW + AJUSTES (2-3 dias)
   ↓
7. LAUNCH + HANDOVER (1 dia)
```

---

## 2. CHECKLIST DE QUALIDADE POR ENTREGÁVEL

### A. DIAGNÓSTICO DIGITAL

**Conteúdo mínimo:**
- [ ] Análise de 5 concorrentes diretos
- [ ] SEO audit (on-page + técnico)
- [ ] Análise de tráfego current (se houver)
- [ ] Audit de campanhas pagas (Google + Meta, se ativo)
- [ ] Análise de presença em redes sociais
- [ ] Score atual (0-100)
- [ ] Top 5 oportunidades
- [ ] Roadmap 90 dias

**Qualidade editorial:**
- [ ] Sem erros ortográficos
- [ ] Insights acionáveis (não óbvios)
- [ ] Dados específicos (não genéricos)
- [ ] Tone profissional mas acessível

**Delivery:**
- [ ] PDF ou Google Doc
- [ ] Máximo 10 páginas
- [ ] Com gráficos/visual
- [ ] Recomendações priorizadas

---

### B. IDENTIDADE VISUAL

**Deliverables:**
- [ ] Paleta de cores (3-5 cores + codes RGB/HEX)
- [ ] Tipografia principal + secundária
- [ ] Logo principal (versões: positive, negative, greyscale)
- [ ] Logo marca (favicon, seal)
- [ ] 30+ mockups (website, cart, envelope, etc)
- [ ] Brand Guidelines (2-3 páginas)

**Qualidade de design:**
- [ ] Coesão visual (cores, formas, estilo)
- [ ] Legibilidade garantida (teste em mobile)
- [ ] Memorabilidade (logo único)
- [ ] Profissionalismo (adequado para setor saúde)
- [ ] Escalabilidade (funciona de 16px a 1000px)

**Padrões técnicos:**
- [ ] Logotipos em PNG + SVG
- [ ] Mockups em alta resolução (300 DPI)
- [ ] Brand guidelines editável (Figma + PDF)
- [ ] Arquivo fonte entregue

---

### C. SITE PREMIUM

**Estrutura:**
- [ ] Homepage + 5-8 subpáginas (especialidades, blog, etc)
- [ ] SEO otimizado (title, meta, heading hierarchy)
- [ ] Mobile responsive (teste em 3+ devices)
- [ ] Acessibilidade WCAG 2.1 AA
- [ ] <3 segundos load time

**Sections obrigatórias:**
1. [ ] Hero + CTA principal
2. [ ] Problema + Solução
3. [ ] Especialidades (com descritivos)
4. [ ] Diferencial/Por que nós
5. [ ] Depoimentos (mín 3, com foto)
6. [ ] FAQ
7. [ ] CTA final + formulário
8. [ ] Footer com contatos

**Qualidade de conteúdo:**
- [ ] Copy sem jargão excessivo
- [ ] Benefício > features
- [ ] Trust signals visíveis (diplomas, anos)
- [ ] Fotos profissionais (consultório + equipe)
- [ ] Sem links quebrados
- [ ] Sem spellcheck errors

**Técnico:**
- [ ] Google Analytics 4 integrado
- [ ] Google Search Console setup
- [ ] SSL certificate
- [ ] Sitemap XML
- [ ] Robots.txt otimizado
- [ ] Open Graph tags para social sharing

**Performance:**
- [ ] PageSpeed Insights >90 desktop
- [ ] PageSpeed Insights >80 mobile
- [ ] Core Web Vitals: Green
- [ ] Compressão de imagens (WebP)

---

### D. LANDING PAGES

**Seções obrigatórias:**
1. [ ] Hero (headline + subheadline + CTA)
2. [ ] Pain point (1-2 parágrafos)
3. [ ] Agitation (opcional, 1 parágrafo)
4. [ ] Solution (3-5 bullets)
5. [ ] Proof (testimonials, stats)
6. [ ] CTA final (form ou link)

**Qualidade de copy:**
- [ ] Headline em 8 palavras máximo
- [ ] Copy conversacional
- [ ] Benefício no topo, não features
- [ ] Prova social (números, depoimentos)
- [ ] Urgência implícita (sem agressividade)

**Design:**
- [ ] Single column layout (mobile-first)
- [ ] CTA acima do fold (visível sem scroll)
- [ ] Contraste alto (AAA WCAG)
- [ ] Botões >44px (mobile tap)
- [ ] Sem distrações (menus, links externos)

**Form:**
- [ ] Máximo 5 campos
- [ ] Auto-fill enabled
- [ ] Error messages claras
- [ ] Confirmação pós-submit

---

### E. SOCIAL MEDIA CONTENT (1 MÊS)

**Quantity:**
- [ ] 16-20 posts (4-5 por semana)
- [ ] Mix de formatos (5 carrosseis + 10 singles + 3 vídeos)

**Copy guidelines:**
- [ ] Máximo 150 caracteres (hook)
- [ ] Hashtags: 5-10 por post
- [ ] Tone: Profissional + humano
- [ ] Sem jargão médico (ou explicado)

**Design:**
- [ ] Padrão visual coeso (mesma fonte, cores)
- [ ] Dimensões corretas (1080x1350 para feed, 1080x1920 para story)
- [ ] Text legível em mobile (mín 14px)
- [ ] Marca visível (logo canto ou watermark)

**Conteúdo:**
- [ ] 40% educativo (tips, técnicas)
- [ ] 40% social proof (depoimentos, antes/depois)
- [ ] 20% promo/engagement (polls, offers)

**Delivery:**
- [ ] Calendário formatado (planilha)
- [ ] Imagens em alta res (.png ou .jpg)
- [ ] Copy pré-escrito (copy+paste ready)
- [ ] Hashtags pré-pesquisadas

---

### F. CRIATIVOS PARA ADS (10-15 PEÇAS)

**Estrutura recomendada:**

**Awareness (3-4 peças):**
- [ ] Educativo (aula curta, tip)
- [ ] Problem-agitation
- [ ] Estatística/dado
- [ ] Video testemunhal (15-30s)

**Consideration (3-4 peças):**
- [ ] Case study antes/depois
- [ ] Depoimento cliente (+ foto)
- [ ] Comparativa vs concorrente
- [ ] Diferencial (3 pontos)

**Conversion (3-4 peças):**
- [ ] Oferta (avaliação grátis, desconto)
- [ ] Urgência (últimas vagas)
- [ ] Garantia ou risco-zero
- [ ] Call-to-action direto

**Retargeting (2-3 peças):**
- [ ] Urgência elevada
- [ ] Prova social forte
- [ ] CTA diferente (whatsapp vs agendamento)

**Qualidade técnica:**
- [ ] Dimensões corretas (Meta: 1200x628, Google: 1200x630)
- [ ] File size <500kb
- [ ] Texto <25% da imagem (Meta policy)
- [ ] Video: MP4, <100MB, com subtitle

**Copywriting ads:**
- [ ] Headline 3 variações mínimo
- [ ] Primary text ≤90 chars
- [ ] Description <30 chars
- [ ] CTA botão apropriado
- [ ] Sem claims infundados
- [ ] Compliance com regras setor saúde

## 3. PADRÕES DE DESIGN

### Tipografia

**Recomendado:**
- Heading: Poppins Bold (ou Inter Bold)
- Subheading: Poppins SemiBold
- Body: Inter Regular
- Button: Poppins SemiBold

**Sizes (responsive):**
- H1: 48px desktop / 32px mobile
- H2: 36px desktop / 24px mobile
- Body: 16px desktop / 14px mobile
- Small: 14px desktop / 12px mobile

### Cores

**Saúde — Paleta recomendada:**
- Primary: Verde/Azul (confiança, saúde)
- Secondary: Complementar (destaque)
- Accent: Verde/Laranja (CTA)
- Neutro: Cinzas (background, text)

**Exemplo santa isabel:**
- Primary: Verde Sage #2A7D6B
- Secondary: Dourado #C4A04A
- Accent: Verde claro #5FA78D
- Neutro: Cinza #F5F5F5, #333333

### Spacing

- Padding padrão: 16px, 24px, 32px, 48px
- Margin padrão: 24px, 32px, 48px, 64px
- Gutter (grid): 24px

### Ícones

- Style: Line icons, 2px stroke
- Tamanho padrão: 24px, 32px, 48px
- Cores: Match primary/secondary

## 4. PADRÕES DE COPYWRITING

### Headlines para Saúde

**Fórmulas comprovadas:**

1. **Benefit-Driven:**
   "Elimine a Dor nas Costas em 3 Semanas"
   "Recupere Mobilidade em 30 Dias"

2. **Problem-Aware:**
   "Tentou Tudo e Nada Funcionou? Existe Solução"
   "Seu Médico Anterior Errou. Veja Por Quê"

3. **Data-Driven:**
   "89% de Sucesso em 2 Meses"
   "5.000+ Pacientes Tratados com Sucesso"

4. **Promise-Based:**
   "Volte a Viver Sem Dor"
   "Seu Movimento Restaurado"

### Subheadlines

**Padrão:**
"[Benefício específico] + [Diferencial] + [Prova social]"

Exemplo:
"Método patenteado, aprovado por 50+ ortopedistas, com garantia de resultado em 3 meses."

### Body Copy

**Estrutura:**
1. Abertura (problema reconhecido)
2. Agitação (consequência)
3. Educação (por que falha)
4. Solução (o que fazemos)
5. Prova (números, depoimentos)
6. CTA

**Principles:**
- Parágrafos curtos (2-3 linhas)
- Frases ativas (evitar passiva)
- Benefício em cada sentença
- Sem jargão (ou explicado)
- Conversacional (você, nós)

- [ ] Cores contrastam bem
- [ ] Sem elementos "pesados"

- [ ] Sem erros ortográficos (x3 leitura)

**Técnico:**
- [ ] Links funcionando
- [ ] Imagens otimizadas
- [ ] Responsive em mobile
- [ ] <3s load (se applicable)

**Compliance:**
- [ ] Sem identidade visual de marca conhecida
- [ ] Sem claims médicos comprovados
- [ ] LGPD: Privacidade clara
- [ ] Acessibilidade básica (contraste, alt text)

## 6. FEEDBACK E AJUSTES

### Rodadas de Review

**Padrão:**
- Round 1: Conceitual (estrutura, messaging)
- Round 2: Refinamento (copy, design details)
- Round 3: Final (ajustes mínimos)
- Launch: Pronto

**SLA de Resposta:**
- Feedback do cliente: 2-3 dias máximo
- Ajustes: 1-2 dias máximo

### Feedback Format

**Feedback útil:**
❌ "Não gostei do design"
✅ "O verde é muito escuro para o setor saúde. Sugiro verde mais claro."

**Solicitar:**
- Feedback específico (seção, elemento)
- Priorização (crítico, importante, nice-to-have)
- Razão (para contexto)

## 9. ERROS COMUNS A EVITAR

### Design
❌ Muitas cores (max 5)
❌ Muitas fontes (max 2-3)
❌ Foto de baixa qualidade
❌ Mobile não otimizado
❌ Falta de contrast (acessibilidade)

### Copywriting
❌ Jargão médico sem explicação
❌ Claims sem fonte
❌ Textos muito longos
❌ Headlines genéricos
❌ Sem CTA claro

### Técnico
❌ Sem SSL
❌ Sem mobile optimization
❌ Imagens não comprimidas
❌ Sem analytics
❌ Links quebrados

### Compliance
❌ Publicidade enganosa
❌ Sem privacidade/LGPD
❌ Sem certificação médica
❌ Sem crédito de depoimentos
