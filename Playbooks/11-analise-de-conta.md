# Playbook 11 — Análise de Conta (Setup de Agentes)

> **Pulso Onboarding Framework** | Etapa 11 de 12

---

## Objetivo

Realizar uma análise profunda da conta do cliente (marketing, negócio, concorrência, narrativa de marca) para alimentar os agentes especializados da Pulso com contexto rico, garantindo execuções cada vez mais precisas e alinhadas.

---

## Quando Executar

- Ao início de cada novo cliente (primeiros 3 dias)
- Na renovação anual de contrato
- Quando houver mudança significativa no negócio do cliente
- Antes de escalar budget de tráfego pago

---

## Framework de Análise (5 Dimensões)

### Dimensão 1 — Negócio

```
1.1 Modelo de negócio
    → Tipo: Clínica própria / Franquia / Consultório PJ / Hospital
    → Faturamento estimado (faixa): até R$50k / R$50–200k / R$200k+/mês
    → Sazonalidade: existe pico em algum mês?
    → Planos de saúde atendidos: [lista]
    → Procedimentos mais lucrativos: [ranking]

1.2 Estrutura da equipe
    → Número de médicos/profissionais
    → Tem recepcionista/secretária?
    → Tem coordenador de marketing interno?
    → Tem gestor de tráfego próprio?

1.3 Tecnologia em uso
    → Sistema de agendamento: [Software]
    → CRM: [Software ou nenhum]
    → Prontuário eletrônico: [Software]
    → WhatsApp Business: SIM / NÃO
```

### Dimensão 2 — Marketing Atual

```
2.1 Histórico de campanhas
    → Já rodou Meta Ads? [SIM/NÃO] → Resultados: [...]
    → Já rodou Google Ads? [SIM/NÃO] → Resultados: [...]
    → Maior aprendizado: [...]
    → Maior fracasso: [...]

2.2 Presença digital atual
    → Instagram: @[handle] | X seguidores | Y% engajamento
    → Google Meu Negócio: X estrelas | Y reviews
    → Site: [URL] | PageSpeed mobile: X | PageSpeed desktop: X
    → YouTube: [existe? URL]
    → TikTok: [existe? URL]

2.3 Orçamento disponível
    → Budget total/mês para marketing: R$ [X]
    → Distribuição ideal: Meta X% / Google X% / Produção X%
```

### Dimensão 3 — Público e ICP

```
3.1 Perfil do Paciente Ideal (ICP)
    → Faixa etária: [X–Y anos]
    → Gênero predominante: [M/F/ambos]
    → Renda familiar: [faixa]
    → Localização: [bairros/cidades]
    → Plano de saúde: [quais]

3.2 Jornada do Paciente
    → Como descobre a clínica? (busca, indicação, social)
    → Tempo entre descoberta e agendamento: [dias]
    → Principal objeção antes de agendar: [...]
    → Por que cancela? [...]
    → Por que retorna? [...]

3.3 Personas documentadas
    → Persona 1: [Nome, idade, perfil, dor, objetivo]
    → Persona 2: [Nome, idade, perfil, dor, objetivo]
```

### Dimensão 4 — Competição

```
4.1 Mapeamento de concorrentes diretos
    | Concorrente | Site | Instagram | GMB | Pontos fortes | Pontos fracos |
    |-------------|------|-----------|-----|---------------|---------------|
    | 1. | | | | | |
    | 2. | | | | | |
    | 3. | | | | | |

4.2 Análise de anúncios da concorrência
    → Meta Ads Library: [captura das peças ativas]
    → Google: [busca pelas palavras-chave principais]
    → Diferenciais que nenhum concorrente comunica: [...]

4.3 Oportunidade de posicionamento
    → Nicho inexplorado: [...]
    → Especialidade sem concorrência local: [...]
    → Canal de aquisição sem concorrência: [...]
```

### Dimensão 5 — Narrativa de Marca

```
5.1 Essência da marca
    → Missão (por que existem?): [...]
    → Visão (onde querem chegar?): [...]
    → Valores (3–5 palavras): [...]
    → Tom de voz: Formal/Informal | Técnico/Acessível | Frio/Quente

5.2 Arquétipo da marca
    → Identificar o arquétipo dominante:
      [ ] Herói (vence o problema)
      [ ] Cuidador (cuida do paciente)
      [ ] Sábio (compartilha conhecimento)
      [ ] Criador (inovação em saúde)
      [ ] Governante (autoridade máxima)

5.3 Proposta de valor única (UVP)
    → Completar: "Para [ICP] que [dor/necessidade],
      [Nome da clínica] é o único [categoria] que [benefício único]
      porque [razão de acreditar]."
```

---

## Output da Análise

### Documento de Briefing de Conta
- Resumo executivo (1 página)
- As 5 dimensões preenchidas
- Top 3 prioridades estratégicas
- Quick wins (pode fazer esta semana)
- Riscos identificados

### Alimentação dos Agentes
Este briefing deve ser carregado no contexto de cada agente Pulso:
- Web Builder: recebe Dimensões 1, 3, 5
- Content Creator: recebe Dimensões 3, 5
- Ads Manager: recebe Dimensões 2, 3, 4
- Sales Strategist: recebe Dimensões 1, 2, 9 (proposta)
